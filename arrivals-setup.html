🛠 Arrivals setup

One-shot provisioning of planning columns on Arrivals
‹ Hub
What this page does: Provisions the new planning columns on the Arrivals board so cleaner assignments, hours, start times, and the linen / hot-tub / dirties drivers live directly on each booking — not on separate Cleans / HT Cleans rows.
  • Idempotent. Re-runs are safe. Columns matched by title — won't duplicate if already there.
  • No data is touched. Existing Arrivals rows keep all current values. New columns start empty.
  • board_relation columns target the Staff board. So renames don't break assignments.

1. Scan the Arrivals board

Reads every column on the Arrivals board to see which planning columns already exist.

2. Provision API-creatable columns

Creates simple-type columns (numbers, hour, long_text) via Monday's API. Click only once — the API queues them.

3. Manual setup — Staff-linked columns

Monday's API no longer permits creating board-relation columns on existing boards. Add these manually on the Arrivals board — takes about 2 minutes.

How to add each one:
  1. Open the Guest Arrivals board on Monday.
  2. Click the + at the rightmost column header → More columns → Connect boards.
  3. Name it exactly as shown below (case-sensitive matching).
  4. Select the Staff board as the connected board.
  5. Save. Repeat for each remaining column.

Once added, come back to this page and click 🔍 Scan board again — they'll show as Exists.
