Staff Shifts — date migration

One-off fix: shifts saved through staff-shifts.html during BST were dated one day too early due to a UTC conversion bug. This page shifts every shift's date forward by 1 day so the data matches what you actually clicked.
What this does, in order:
  • Fetches every row on the Staff Shifts Monday board (id 5095155773)
  • For each, computes new_date = date + 1 day
  • Pushes batched change_multiple_column_values mutations
  • Logs every batch's success/failure to the panel below
⚠ Run only ONCE. Running twice would shift dates by 2 days. This page records a marker in your browser's localStorage after a successful run and refuses to run again from the same browser. If you ever need to revert, the marker also lets a "rollback −1 day" button appear.
